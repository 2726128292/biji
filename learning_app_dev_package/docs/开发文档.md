# 知行笔记：离线费曼笔记与镜像错题本学习软件开发文档

版本：V1.0  
日期：2026-06-06  
目标：构建一个无后端、离线优先、可安装为 PWA 的个人学习软件。

## 1. 产品定位

本项目包含两个核心模块：

1. **费曼笔记**：使用目录树、CodeMirror 编辑器、费曼模板、高亮区块、双向链接和标签辅助理解知识。
2. **背题刷题**：使用结构化题库、可视化题目编辑器、背题模式、刷题模式和“镜像错题本”完成重复筛选学习。

项目不提供注册登录、强制云同步、社交、广告、收费、复杂统计图表，也不依赖在线服务。

## 2. 已确认的关键设计决策

### 2.1 题库不以 Markdown 文件作为核心交互

用户界面中不出现 `.md` 文件扩展名。题库由以下结构组成：

```text
题库
└─ 章节文件夹（允许无限嵌套）
   └─ 题目（结构化数据）
```

用户通过可视化表单新建、编辑和删除题目。Markdown、TXT、Word 仅作为可选的批量导入和导出格式。

### 2.2 错题本是镜像筛选视图

错题本与原题库目录结构一一对应：

- 原题在哪一本题库、哪一个章节、哪一个子章节中，错题本中仍然显示在相同位置。
- 错题本不复制题目正文，只保存对原题的引用。
- 原题修改后，错题本同步更新。
- 没有错题的目录默认隐藏，可切换为“显示完整结构”。

### 2.3 每次刷题前由用户选择目标错题本

用户开始背题或刷题前，必须选择以下一种处理方式：

```text
不会或答错的题目：
● 新建错题本，并输入名称
○ 加入已有错题本
○ 不加入任何错题本
```

每一次生成的错题本永久保留，不覆盖上一轮结果。任何错题本都可以再次作为来源重复刷题，继续筛选出更少的题目。

示例：

```text
原始题库（500）
→ 第一轮不会（126）
→ 第二轮不会（43）
→ 考前重点（11）
→ 最后再背一次（3）
```

### 2.4 “不认识”与“答错”均可进入目标错题本

背题模式中点击“❌ 不认识”，刷题模式中点击“不会”或提交错误答案，题目均加入本次选择的目标错题本。若用户选择“不加入任何错题本”，只记录本次练习结果。

## 3. 技术架构

```text
Vite

TypeScript
Pinia
```

部署方式：通过 GitHub Actions 自动执行 `npm run build` 并发布 `dist` 到 GitHub Pages。

## 4. PWA 与数据安全

- 应用可安装到桌面或主屏幕；不同浏览器的安装入口不同，不能保证首次访问必然弹出安装提示。
- 所有业务数据存储在 IndexedDB，不上传服务器。
- 应用启动后调用 `navigator.storage.persist()` 申请持久化存储，但浏览器可能不批准。
- 必须提供完整 ZIP / JSON 备份和恢复。
- 更新 Service Worker 时，后台下载新版本；不强制刷新正在编辑的页面。

## 5. 响应式布局

### 桌面端

- 屏幕宽度 `>= 768px`。
- 左侧目录默认展开，初始宽度 240px，可调整至 150-350px。
- 顶部导航栏固定，高度 54px。
- 支持右键菜单和拖拽排序。

### 手机端

- 屏幕宽度 `< 768px`。
- 左侧目录默认收起，通过 ☰ 滑出。
- 使用节点右侧 `⋮` 和长按打开底部操作菜单，不依赖右键。
- 移动题目或章节时提供“移动到……”目录选择器。

## 6. 费曼笔记模块

### 6.1 目录

根节点为“我的笔记”。支持无限层级文件夹、创建笔记、重命名、删除、移动、排序、展开、折叠、导出。

### 6.2 编辑器

- 显示当前笔记完整路径。
- 右上角提供“插入费曼模板”。
- 模板包含：问题、重点、解释、卡壳点、类比、总结。
- CodeMirror 实时识别关键字区块并应用背景色。
- 输入停止 500ms 后写入 IndexedDB；切换页面或退出前强制保存。
- 每 30 秒在内容变化时生成历史快照，每个文件最多 10 个。

### 6.3 Obsidian 风格能力

- `[[笔记名称]]` 双向链接，内部保存笔记 ID，避免重命名失效。
- `#标签` 自动识别。
- 笔记底部显示反向链接。

## 7. 题库模块

### 7.1 结构

```text
我的题库
└─ 题库
   └─ 章节文件夹（无限嵌套）
      └─ 结构化题目
```

支持新建题库、删除题库、重命名题库、导入、导出、章节管理、题目管理。

### 7.2 题目类型

- 单选题
- 多选题
- 判断题
- 填空题
- 简答题

题目使用可视化表单创建，不要求用户编辑 Markdown。

## 8. 背题与刷题

### 8.1 背题模式

每张卡片显示题目、选项、折叠答案、折叠解析，并提供：

```text
✅ 认识
❌ 不认识
⭐ 重点
```

点击“不认识”时，根据本次设置加入目标错题本。

### 8.2 刷题模式

所有题型统一提供“不会”按钮。答错或点击“不会”后：

1. 显示正确答案和解析。
2. 标记本题为 `wrong` 或 `unknown`。
3. 若已选择目标错题本，按原题位置添加题目引用。
4. 同一道题在同一错题本中只保留一次。

## 9. 镜像错题本

### 9.1 目标

让题目通过重复刷题逐渐减少，同时保留每轮结果和原始章节位置。

### 9.2 规则

- 每个错题本有名称、创建时间、来源题库或来源错题本。
- 用户可从原始题库或任意错题本开始重复刷题。
- 新错题本内部动态镜像原题库结构。
- 上一轮错题本永久保留。
- 用户可以选择将错题加入已有错题本，合并多个来源。
- 用户也可以选择不生成任何错题本。

### 9.3 原题删除

删除已被错题本引用的原题时提示：

```text
● 同时从所有错题本移除
○ 保留错题快照
○ 取消删除
```

## 11. 备份与导出

### 完整备份

ZIP / JSON 必须包含：笔记、题库、章节、题目、错题本、引用关系、练习历史、历史版本、标签、链接、设置和 schemaVersion。

### 文档导出

支持 TXT、Markdown 和 Word。Word 尽量保留标题层级、关键字背景色和基础排版，但不承诺在不同办公软件中完全一致。
